## Mobile Interactions

### Gestures
- **Pull to Refresh**: Updates shipment data
- **Scroll**: Timeline and items list
- **Tap**: Select customer, product, action buttons
- **Long Press**: (future) Quick actions menu

### Loading States
All mutations show:
1. Button spinner
2. Disabled state on other buttons
3. Global loading overlay (from API interceptor)

### Success/Error Feedback
- Success: Green dialog with checkmark
- Error: Red dialog with detailed message
- Confirm: Yellow dialog with details list

---

## Responsive Design Notes

### Breakpoints
- Small phones (< 360dp): Single column, compact spacing
- Standard phones (360-600dp): Standard layout
- Tablets (> 600dp): Could show side-by-side (future)

### Touch Targets
- All buttons: Minimum 44x44 points
- List items: Full-width tap area
- Input fields: 48dp height minimum

## Accessibility Considerations

### Screen Reader Support
- All buttons have descriptive labels
- Status badges announce status + label
- Timeline events read in chronological order
- Form fields have proper labels

### Color Contrast
- All text meets WCAG AA (4.5:1 minimum)
- Status colors have sufficient contrast
- Disabled states clearly distinguishable

### Keyboard Navigation
- Tab order follows visual flow
- Enter/Space activate buttons
- Escape closes modals
